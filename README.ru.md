# geolm — узкие геометрические модели: обучил под задачу, выкатил в инференс

`geolm` обучает маленькую нейросеть под одну конкретную задачу вида «вход, затем ответ»
(калькулятор, нормализация строк, классификация, преобразование форматов) и сразу
даёт готовый инференс: библиотеку на Python, командную строку и HTTP-сервис.

Внутри не трансформер, а геометрическая модель: каждый токен — обучаемая точка в
пространстве малой размерности, последовательность читается рекуррентным движком как
траектория по точкам, а следующий токен выбирается по близости к предсказанной точке.
Одно облако точек работает и на входе, и на выходе, поэтому модель компактна, а
стоимость длинного входа растёт линейно, и память при генерации постоянна.

Зачем это, когда есть трансформеры: в парном замере на одинаковой задаче и при
одинаковом размере (2,6 млн параметров, обучение в равных условиях) геометрическая
модель дала то же качество (91,5% против 91,6% в допуске), но отвечала втрое быстрее
(0,03 мс против 0,10 мс на запрос) и держала почти вчетверо большую пропускную
способность (38 500 против 10 300 запросов в секунду на одной карте). Ниша: узкие
высоконагруженные сервисы, потоковая обработка, слабое железо.

## Установка

```bash
cd geolm
pip install -e .
```

Единственная зависимость — PyTorch. Работает на процессоре; если есть CUDA, включается сама.

## Быстрый старт за 3 шага

Шаг 1. Подготовьте данные в формате JSONL: одна строка — один пример, поля `input` и `output`.

```json
{"input": "357+468=", "output": "825"}
{"input": "999-1=",   "output": "998"}
```

Шаг 2. Обучите модель одной командой:

```bash
geolm train --data examples/data.jsonl --out task.geolm --steps 20000
```

Шаг 3. Пользуйтесь:

```bash
geolm predict --model task.geolm --input "357+468="     # разовый ответ
geolm eval    --model task.geolm --data holdout.jsonl   # точность на отложенных данных
geolm serve   --model task.geolm --port 8080            # HTTP-сервис
curl -X POST localhost:8080/predict -d '{"input": "357+468="}'
```

## Из Python (интерфейс в духе scikit-learn)

```python
from geolm import GeoLM

pairs = [("357+468=", "825"), ("999-1=", "998"), ...]   # (вход, выход)

model = GeoLM.new(pairs, dim=128, hidden=512)   # словарь строится из данных сам
model.fit(pairs, steps=20000)                   # обучение с оценкой и лучшим чекпойнтом
model.save("task.geolm")                        # один файл: веса + конфиг + словарь

model = GeoLM.load("task.geolm")
model.predict("357+468=")                       # "825"
model.predict_batch(["1+1=", "2+2="])           # пакетно, быстро
model.evaluate(holdout_pairs)                   # % точных совпадений
```

Полный рабочий пример: `python examples/calculator.py` — генерирует данные,
обучает калькулятор, показывает ответы и сохраняет `calc.geolm`.

## Как выбрать размер модели

| Задача | dim | hidden | layers | steps |
|---|---|---|---|---|
| Простое преобразование строк | 48–96 | 160–384 | 1–2 | 3–10 тыс. |
| Арифметика, форматы, извлечение | 128 | 512 | 2 | 20–40 тыс. |
| Сложные многошаговые ответы | 256 | 1024 | 2–4 | 40+ тыс. |

Правило: если точность на отложенных данных ещё растёт к концу обучения — добавьте шагов;
если обучающая точность высокая, а отложенная нет — добавьте данных.

## Формат файла модели

`*.geolm` — это один файл `torch.save` со словарём, конфигурацией и весами.
Никаких внешних файлов не нужно: модель переносится копированием одного файла.

## Ограничения (честно)

- Это специалист под одну задачу, а не универсальный ассистент. Для широкой эрудиции
  и сложных рассуждений используйте большие трансформеры.
- Точная арифметика с делением до многих знаков даётся плохо (как и трансформеру
  того же размера).
- Словарь посимвольный и строится из обучающих данных: символы, которых не было
  в данных, модель не знает.
