# Как протестировать geolm: пошаговое руководство

Это руководство рассчитано на человека, который не разбирается в машинном обучении.
Каждый шаг содержит команду, ожидаемый результат и пояснение, что именно этот шаг
доказывает. Все команды выполняются в терминале из папки проекта `geolm`.

Проверка устроена по уровням, от простого к полному: установка, самопроверка,
командная строка, сетевой сервис, качество на своих данных, скорость. Если все
уровни пройдены, система полностью работоспособна.

---

## Уровень 0. Подготовка

Откройте терминал и перейдите в папку проекта:

```bash
cd ~/Desktop/geolm
```

Установите пакет (достаточно одного раза):

```bash
python3 -m pip install --user -e .
```

Если команда `geolm` после установки не находится, добавьте путь к установленным
скриптам в текущую сессию терминала:

```bash
export PATH="$HOME/Library/Python/3.9/bin:$PATH"
```

Проверка, что установка удалась:

```bash
python3 -c "import geolm; print('geolm', geolm.__version__)"
```

Ожидаемый результат: строка `geolm 0.1.0`. Это доказывает, что библиотека
установлена и импортируется.

---

## Уровень 1. Самопроверка (одна команда, около минуты)

```bash
python3 tests/test_smoke.py
```

Ожидаемый результат, строка вида:

```
OK: обучение работает, точность 75.0%, сохранение/загрузка/инференс целы
```

Что здесь происходит: тест обучает крошечную модель разворачивать строки задом
наперёд, затем сохраняет её в файл, загружает обратно и сверяет ответы. Если тест
прошёл, значит работоспособен весь цикл: обучение, сохранение, загрузка и ответы.
Точность выше шестидесяти процентов считается успехом (модель нарочно крошечная,
чтобы тест шёл минуту, а не час).

Если тест упал с ошибкой про отсутствие модуля torch, установите PyTorch:
`python3 -m pip install --user torch`.

## Уровень 2. Полный пример: калькулятор (несколько минут)

```bash
python3 examples/calculator.py
```

Ожидаемый результат: строки прогресса обучения с растущей точностью, затем
проверочные ответы и сообщение о сохранении модели:

```
[geolm] шаг 6000/6000  loss=...  точное совпадение=9X.X%  <- лучшее
  357+468=825
  999-1=998
модель сохранена: calc.geolm
```

Что это доказывает: библиотека способна с нуля обучить настоящую рабочую модель
под задачу (сложение и вычитание) и модель отвечает верно. После этого шага в папке
появится файл `calc.geolm`, он понадобится на следующих уровнях.

---

## Уровень 3. Командная строка

Все проверки этого уровня используют модель `calc.geolm` из уровня 2.

Разовый ответ:

```bash
geolm predict --model calc.geolm --input "357+468="
```

Ожидаемый результат: `825`.

Пакетный режим (несколько вопросов за раз, по одному на строку):

```bash
printf "1+1=\n40+50=\n999-1=\n" | geolm predict --model calc.geolm
```

Ожидаемый результат: три строки с ответами 2, 90 и 998.

Измерение точности на файле с правильными ответами:

```bash
geolm eval --model calc.geolm --data examples/data.jsonl
```

Ожидаемый результат: строка вида `[geolm] точных совпадений: 9X.X% на 3000 примерах`.
Что это доказывает: модель можно проверить на любом наборе данных с известными
ответами и получить объективное число, а не впечатление.

## Уровень 4. Сетевой сервис (так модель встраивают в приложения)

Запустите сервис в первом окне терминала:

```bash
geolm serve --model calc.geolm --port 8080
```

Ожидаемый результат: строка `[geolm] сервис на http://0.0.0.0:8080  (POST /predict)`,
после чего терминал остаётся занят сервисом, это нормально.

Во втором окне терминала отправьте запросы:

```bash
curl -X POST localhost:8080/predict -d '{"input": "357+468="}'
curl -X POST localhost:8080/predict -d '{"inputs": ["1+1=", "40+50="]}'
```

Ожидаемый результат: `{"output": "825"}` на первый запрос и
`{"outputs": ["2", "90"]}` на второй. Что это доказывает: модель доступна по сети
как обычный сервис, и к ней может обращаться любое приложение на любом языке.

Остановить сервис: нажать Ctrl+C в первом окне.

## Уровень 5. Качество на своих данных

Подготовьте два файла в формате JSONL (одна строка, один пример, поля input и
output): `train.jsonl` для обучения и `holdout.jsonl` для честной проверки.
Важно: примеры из проверочного файла не должны встречаться в обучающем, иначе
оценка будет завышенной (модель могла их просто запомнить).

```bash
geolm train --data train.jsonl --out my_task.geolm --steps 20000
geolm eval  --model my_task.geolm --data holdout.jsonl
```

Как читать результат. Точность на holdout, это главная цифра качества: доля
вопросов, на которые модель ответила в точности правильно. Правила настройки:
если точность в конце обучения ещё росла от шага к шагу, увеличьте `--steps`;
если на обучении точность высокая, а на holdout низкая, добавьте разнообразных
данных; если задача сложная, увеличьте размер модели (`--dim 256 --hidden 1024`).

## Уровень 6. Скорость (по желанию)

Замер скорости ответов на тысяче запросов:

```bash
python3 - <<'EOF'
import time
from geolm import GeoLM
model = GeoLM.load("calc.geolm")
qs = [f"{i%900}+{(i*7)%900}=" for i in range(1000)]
t0 = time.time()
model.predict_batch(qs)
dt = time.time() - t0
print(f"1000 ответов за {dt:.2f} с, то есть {1000/dt:.0f} запросов в секунду")
EOF
```

Ожидаемый результат: число запросов в секунду. На обычном процессоре ноутбука
это сотни и тысячи запросов в секунду, на видеокарте десятки тысяч.

## Частые проблемы

Команда `geolm` не найдена. Причина: папка со скриптами не в PATH. Решение:
`export PATH="$HOME/Library/Python/3.9/bin:$PATH"`, либо вызывать через
`python3 -m geolm.cli`.

Установка падает на `pip install -e .`. Причина: старый pip. Решение:
`python3 -m pip install --user --upgrade pip` и повторить установку.

Обучение идёт, но точность около нуля. Причины по частоте: мало шагов (увеличьте
`--steps`), слишком мало или однообразные данные (нужны тысячи разнообразных
примеров), в проверочных данных встречаются символы, которых не было в обучающих
(модель знает только символы из обучения).

Ответы обрезаются. Причина: ответ длиннее предела генерации. Решение: обучить с
большим `--max-new` (по умолчанию 64 символа).
